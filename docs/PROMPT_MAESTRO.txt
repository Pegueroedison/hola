REQUISITO DE PUBLICACIÓN UNIVERSAL Y ALTO RENDIMIENTO:
- La entrega final debe ser una versión compilada, optimizada y lista para producción.
- Debe poder publicarse sin pantalla blanca en GitHub Pages, Netlify, Vercel, hosting tradicional, dominio propio y subcarpetas.
- La versión publicada debe usar rutas relativas y no depender de una ruta fija del repositorio.
- El usuario debe poder subir los archivos compilados directamente al hosting y ver la interfaz sin ejecutar comandos.
- El código fuente completo debe incluirse aparte para futuras modificaciones, pero la raíz de la entrega debe contener la app compilada lista para publicar.
- La interfaz debe funcionar correctamente en móvil, tablet, PC, PWA instalable y Desktop Windows mediante Tauri.
- La app no debe quedarse en blanco si Supabase, R2 u otras variables todavía no están configuradas; debe mostrar la interfaz y avisos de configuración pendientes.
- La compilación debe priorizar velocidad, carga rápida, assets optimizados, división razonable de código y compatibilidad de despliegue.
